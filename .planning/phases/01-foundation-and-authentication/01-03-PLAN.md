  <files>
    Dockerfile, docker-compose.yml, docker/nginx.conf, docker/entrypoint.sh, .dockerignore
  </files>
  <action>
    1. Create `.dockerignore` with: node_modules, dist, .git, .planning, *.md, .env*.

    2. Create `Dockerfile` per RESEARCH.md Docker example:
       - Stage 1 (build): FROM node:22-alpine, WORKDIR /app, COPY package*.json, RUN npm ci, COPY . ., RUN npm run build.
       - Stage 2 (serve): FROM nginx:alpine, COPY --from=build /app/dist to /usr/share/nginx/html, COPY docker/nginx.conf to /etc/nginx/conf.d/default.conf, COPY docker/entrypoint.sh, chmod +x, EXPOSE 80, ENTRYPOINT + CMD.

    3. Create `docker/nginx.conf` per RESEARCH.md:
       - listen 80
       - root /usr/share/nginx/html
       - SPA routing: `location / { try_files $uri $uri/ /index.html; }`
       - API reverse proxy: `location /api/ { proxy_pass http://app:8080; }` with proxy headers (Host, X-Real-IP, X-Forwarded-For, X-Forwarded-Proto)
       - Static asset caching: 1y expiry with immutable for js/css/images/fonts
       - Gzip: on, for text/css, application/javascript, application/json, image/svg+xml
       - Security headers: X-Frame-Options SAMEORIGIN, X-Content-Type-Options nosniff, Referrer-Policy strict-origin-when-cross-origin

    4. Create `docker/entrypoint.sh` per RESEARCH.md:
       - Replace __API_BASE_URL__, __KEYCLOAK_URL__, __KEYCLOAK_REALM__, __KEYCLOAK_CLIENT_ID__ placeholders in index.html using sed
       - Use environment variable defaults: API_BASE_URL:-/api/v1, KEYCLOAK_URL:-http://localhost:8180, KEYCLOAK_REALM:-cookbook, KEYCLOAK_CLIENT_ID:-cookbook-frontend
       - exec "$@" at end to pass control to nginx

    5. Create `docker-compose.yml`:
       - Service "frontend" building from current directory
       - Port mapping: 3000:80
       - Environment variables: API_BASE_URL, KEYCLOAK_URL, KEYCLOAK_REALM, KEYCLOAK_CLIENT_ID (with defaults)
       - Network: cookbook-network as external (created by backend's docker-compose)
       - Depends_on is NOT needed since the backend runs separately

    6. Verify Docker image builds: `docker build -t cookbook-frontend .`
  </action>
