  <action>
    Create the feature directory structure and all type/API/query files.

    **src/features/recipes/api/types.ts:**
    - `SpringPage<T>` generic: content, number, size, totalElements, totalPages, last, first
    - `Tag`: id (number), name, parentId (number | null)
    - `TagNode`: id, name, children (TagNode[]) -- for tree rendering
    - `RecipeSummary`: id (string), title, imageUrl (string | null), cookTimeMinutes (number), averageRating (number), tags (Tag[]), isFavorite (boolean)
    - `RecipeDetail`: extends summary with description, servings (number), prepTimeMinutes, ingredients (RecipeIngredient[]), steps (RecipeStep[]), ratings summary
    - `RecipeIngredient`: id, amount (number), unit (string), ingredientName (string), ingredientId (number)
    - `RecipeStep`: id, stepNumber (number), instruction (string)
    - `RecipeFilters`: q? (string), tagIds? (number[])
    - `buildTagTree(tags: Tag[]): TagNode[]` utility function that converts flat tag list with parentId to nested tree

    **src/features/recipes/api/recipe-api.ts:**
    - `getRecipes(params: RecipeFilters & { page: number; size?: number })` -> GET `recipes` with searchParams (page, size default 12, search from q, tagIds joined with comma). Returns `SpringPage<RecipeSummary>`
    - `getRecipe(id: string)` -> GET `recipes/${id}`. Returns `RecipeDetail`
    - `getTags()` -> GET `tags`. Returns `Tag[]`

    **src/features/recipes/api/recipe-queries.ts:**
    - `recipeQueries.all()` -> `['recipes']`
    - `recipeQueries.list(filters: RecipeFilters)` -> infiniteQueryOptions with queryKey `['recipes', 'list', filters]`, initialPageParam 0, getNextPageParam: `(lastPage) => lastPage.last ? undefined : lastPage.number + 1`
    - `recipeQueries.detail(id: string)` -> queryOptions with queryKey `['recipes', 'detail', id]`
    - `recipeQueries.tags()` -> queryOptions with queryKey `['tags']`

    **src/features/ratings/api/types.ts:**
    - `RatingCriterion`: id (number), name (string), description? (string)
    - `RatingSummary`: overallAverage (number), criteriaAverages ({ criterionId: number, criterionName: string, average: number }[]), totalRatings (number)
    - `UserRating`: id (number), scores ({ criterionId: number, score: number }[]), createdAt (string)
    - `RatingSubmission`: scores ({ criterionId: number, score: number }[])

    **src/features/ratings/api/rating-api.ts:**
    - `getRatingCriteria()` -> GET `rating-criteria`. Returns `RatingCriterion[]`
    - `getRecipeRatings(recipeId: string)` -> GET `recipes/${recipeId}/ratings`. Returns `RatingSummary`
    - `getUserRating(recipeId: string)` -> GET `recipes/${recipeId}/ratings/me`. Returns `UserRating | null`
    - `submitRating(recipeId: string, data: RatingSubmission)` -> POST `recipes/${recipeId}/ratings`
    - `updateRating(recipeId: string, ratingId: number, data: RatingSubmission)` -> PUT `recipes/${recipeId}/ratings/${ratingId}`
    - `deleteRating(recipeId: string, ratingId: number)` -> DELETE `recipes/${recipeId}/ratings/${ratingId}`

    **src/features/ratings/api/rating-queries.ts:**
    - `ratingQueries.criteria()` -> queryOptions for criteria list
    - `ratingQueries.summary(recipeId: string)` -> queryOptions for recipe rating summary
    - `ratingQueries.userRating(recipeId: string)` -> queryOptions for current user's rating

    **src/features/favorites/api/favorite-api.ts:**
    - `getFavorites()` -> GET `users/me/favorites`. Returns `RecipeSummary[]`
    - `toggleFavorite(recipeId: string)` -> POST `users/me/favorites/${recipeId}`
    - `removeFavorite(recipeId: string)` -> DELETE `users/me/favorites/${recipeId}`

    **src/features/favorites/api/favorite-queries.ts:**
    - `favoriteQueries.list()` -> queryOptions for favorites list
    - `useToggleFavorite()` -> useMutation with optimistic update (cancel queries, snapshot, optimistic toggle, rollback on error, invalidate on settled). Follow Pattern 5 from research.

    **src/features/profile/api/types.ts:**
    - `UserProfile`: id (string), displayName (string), email (string)
    - `UserPreferences`: allergenIds (number[]), favoriteIngredientIds (number[]), hiddenTagIds (number[])
    - `Allergen`: id (number), name (string)

    **src/features/profile/api/profile-api.ts:**
    - `getProfile()` -> GET `users/me`. Returns `UserProfile`
    - `updateProfile(data: { displayName: string })` -> PUT `users/me`
    - `getPreferences()` -> GET `users/me/preferences`. Returns `UserPreferences`
    - `updatePreferences(data: Partial<UserPreferences>)` -> PUT `users/me/preferences`
    - `getAllergens()` -> GET `allergens`. Returns `Allergen[]`

    **src/features/profile/api/profile-queries.ts:**
    - `profileQueries.profile()`, `profileQueries.preferences()`, `profileQueries.allergens()`

    **src/features/ingredients/api/types.ts:**
    - `Ingredient`: id (number), name (string), allergenIds (number[]), nutritionalInfo? (string)
    - `IngredientDetail`: extends Ingredient with description? (string), allergenNames (string[])

    **src/features/ingredients/api/ingredient-api.ts:**
    - `getIngredients(params?: { search?: string })` -> GET `ingredients` with optional search param
    - `getIngredient(id: number)` -> GET `ingredients/${id}`. Returns `IngredientDetail`

    **src/features/ingredients/api/ingredient-queries.ts:**
    - `ingredientQueries.list(search?: string)`, `ingredientQueries.detail(id: number)`

    All API functions import `apiClient` from `@/shared/api/client`. All use `.json<Type>()` for response typing. Use `ky` searchParams option (not URLSearchParams constructor) for query parameters.
