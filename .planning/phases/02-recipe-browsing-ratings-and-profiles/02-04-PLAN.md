<task type="auto" tdd="true">
  <name>Task 1: Profile page with display name form, allergen preferences, favorite ingredients, and tag visibility</name>
  <files>
    src/routes/_authenticated/profile.tsx,
    src/features/profile/ui/ProfileForm.tsx,
    src/features/profile/ui/AllergenPreferences.tsx,
    src/features/profile/ui/FavoriteIngredients.tsx,
    src/features/profile/ui/TagVisibility.tsx,
    src/features/profile/ui/ProfileForm.test.tsx
  </files>
  <behavior>
    - ProfileForm renders input with current display name, save button submits update
    - ProfileForm save button disabled when name unchanged or empty
    - AllergenPreferences renders toggle switches for each allergen
    - FavoriteIngredients shows current favorites with remove buttons and search to add new
    - TagVisibility shows tags with hide/unhide toggles
  </behavior>
  <action>
    **ProfileForm.tsx:**
    - Props: `{ profile: UserProfile; onSave: (data: { displayName: string }) => void; isSaving: boolean }`
    - Local state for `displayName` initialized from `profile.displayName`
    - Input: GlassPanel-wrapped text input with label "Display Name"
    - Save button: `bg-accent/20 text-accent-light hover:bg-accent/30 px-4 py-2 rounded-lg transition-colors disabled:opacity-50`
    - Disabled when `displayName === profile.displayName || displayName.trim() === '' || isSaving`
    - On submit (form onSubmit, prevent default): call `onSave({ displayName: displayName.trim() })`
    - Show success feedback: brief "Saved!" text that fades after 2 seconds (local state)

    **AllergenPreferences.tsx:**
    - Props: `{ allergens: Allergen[]; selectedIds: number[]; onToggle: (allergenId: number) => void; isLoading: boolean }`
    - If isLoading: skeleton lines
    - Renders list of allergens in a GlassPanel, each as a row with:
      - Allergen name in `text-white/80`
      - Toggle switch on right: `button` styled as sliding toggle (rounded-full, bg-accent when active, bg-white/20 when inactive, with inner circle that slides)
    - On toggle: call `onToggle(allergenId)` -- parent handles mutation

    **FavoriteIngredients.tsx:**
    - Props: `{ favoriteIds: number[]; onAdd: (ingredientId: number) => void; onRemove: (ingredientId: number) => void }`
    - Fetch ingredients list using `useQuery(ingredientQueries.list())` to get names for favoriteIds
    - Search input at top (use useDebounce) to find ingredients to add
    - Current favorites shown as chips with X to remove
    - Search results shown as a dropdown/list below search input, each with "+" button to add
    - Filter out already-favorited ingredients from search results
    - Wrap in GlassPanel

    **TagVisibility.tsx:**
    - Props: `{ hiddenTagIds: number[]; onToggle: (tagId: number) => void }`
    - Fetch all tags using `useQuery(recipeQueries.tags())`
    - Render flat list of all tags (not tree) in GlassPanel, each row:
      - Tag name
      - Eye/EyeOff icon toggle: Eye (visible, default), EyeOff (hidden)
      - Hidden tags shown with `opacity-50` and EyeOff icon
    - On toggle: call `onToggle(tagId)` -- parent handles adding/removing from hiddenTagIds

    **profile.tsx route:**
    - Use `createFileRoute('/_authenticated/profile')` with component `ProfilePage`
    - Fetch: `useQuery(profileQueries.profile())`, `useQuery(profileQueries.preferences())`, `useQuery(profileQueries.allergens())`
    - Mutations:
      - `updateProfileMutation = useMutation({ mutationFn: updateProfile, onSuccess: invalidate profile queries })`
      - `updatePreferencesMutation = useMutation({ mutationFn: updatePreferences, onSuccess: invalidate preferences queries })`
    - Layout: page title "My Profile" (h1), then sections stacked vertically with gap-6:
      1. Profile section: ProfileForm with current profile data
      2. Allergen Preferences section: heading "Allergen Preferences" + AllergenPreferences
      3. Favorite Ingredients section: heading "Favorite Ingredients" + FavoriteIngredients
      4. Tag Visibility section: heading "Tag Visibility" + TagVisibility
    - Each section heading: `h2.text-xl.font-semibold.text-white.mb-3`
    - Allergen toggle handler: compute new allergenIds array (add if not present, remove if present), call `updatePreferencesMutation.mutate({ allergenIds: newIds })`
    - Favorite ingredient add/remove: similarly compute new favoriteIngredientIds, call updatePreferences
    - Tag visibility toggle: compute new hiddenTagIds, call updatePreferences
    - Loading state: skeleton sections

    **ProfileForm.test.tsx:**
    - Test: renders input with current display name
    - Test: save button disabled when name unchanged
    - Test: save button disabled when name is empty/whitespace
    - Test: calls onSave with trimmed name on submit
    - Test: shows "Saved!" feedback after successful save
    - Use renderWithProviders from test/utils.tsx
  </action>

    - IngredientCard displays name, allergen badges, and nutritional info

    - Empty search returns full list
  </behavior>
  <action>
    **IngredientSearch.tsx:**
    - Props: `{ value: string; onChange: (v: string) => void }`
    - Same pattern as SearchBar from recipes: local state + useDebounce(300ms) + GlassPanel with Search icon
    - Placeholder: "Search ingredients..."
    - Clear button (X icon) when input non-empty

    **IngredientCard.tsx:**
    - Props: `{ ingredient: Ingredient; onSelect: (id: number) => void; isExpanded: boolean; detail?: IngredientDetail }`
    - Default (collapsed): GlassCard with ingredient name and allergen count badge if any allergens
    - Expanded: show full detail below name:
      - Allergen badges: each allergen name in a `span.text-xs.px-2.py-0.5.rounded-full.bg-red-500/20.text-red-300` chip
      - Nutritional info (if present): `p.text-sm.text-white/60` with the nutritional text
      - Description (if present): `p.text-sm.text-white/60.mt-2`
    - On click: call `onSelect(ingredient.id)` to toggle expansion

    **ingredients.tsx route:**
    - Use `createFileRoute('/_authenticated/ingredients')` with component `IngredientsPage`
    - Local state: `search` string, `expandedId` (number | null)
    - Debounced search passed to `useQuery(ingredientQueries.list(debouncedSearch))`
    - When expandedId is set: `useQuery(ingredientQueries.detail(expandedId))` for the expanded ingredient
    - Layout: page title "Ingredients" (h1), IngredientSearch below, then ingredient list
    - Render list in a vertical stack (`div.space-y-3`) of IngredientCard components
    - Loading: skeleton cards (4 rectangular skeletons)
    - Empty state: EmptyState with Wheat icon, "No ingredients found", "Try a different search term"
    - On card click: toggle expandedId (if same id clicked again, collapse)

    - Test: shows clear button when input non-empty
    - Test: calls onChange after debounce
    - Use vi.useFakeTimers for debounce testing, renderWithProviders from test/utils
