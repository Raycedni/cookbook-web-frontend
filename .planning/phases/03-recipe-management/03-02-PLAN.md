**RecipeWizard.tsx** — The 2-step wizard container:
- Props: `{ initialData?: WizardState; recipeId?: string; mode: 'create' | 'edit' }`
- State: `const [form, setForm] = useState<WizardState>(initialData ?? defaultWizardState)`
- `const [currentStep, setCurrentStep] = useState(0)` (0 = basics, 1 = details)
- `const [isSaving, setIsSaving] = useState(false)`
- Partial updater: `function updateForm(partial: Partial<WizardState>) { setForm(prev => ({ ...prev, ...partial })) }`
- Fetch units via `useQuery(recipeQueries.units())`
- Progress indicator at top: two segments showing "Step 1: Basics" and "Step 2: Details" with active/completed states. Use glassmorphism styling — active step has accent border/bg, completed has checkmark.
- Render WizardStepBasics when currentStep=0, WizardStepDetails when currentStep=1
- Step 1 validation for "Next": name is required (non-empty), at least 1 ingredient with ingredientId set
- Save handler (called from Step 2):
  1. Build CreateRecipeRequest or UpdateRecipeRequest from form state — map `name` (not title), step `description` (not instruction), ingredients with orderIndex
  2. Call createRecipe or updateRecipe depending on mode
  3. Upload hero image if form.heroImage exists: `await uploadRecipeImage(recipe.id, form.heroImage, true)`
  4. Upload step images sequentially: for each step in response, find matching form step by index, upload each file via `uploadStepImage(recipe.id, step.id, file)`
  5. Invalidate `recipeQueries.all()` query key
  6. Navigate to `/recipes/$recipeId` with the returned recipe ID
  7. Show isSaving state on save button
- defaultWizardState: empty name, empty description, servings=4, prepTimeMinutes=0, cookTimeMinutes=0, no heroImage, empty ingredients array, empty steps array, empty tagIds
