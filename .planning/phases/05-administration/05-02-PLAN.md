**UserManagement.tsx:**
- Uses `useQuery(adminQueries.users(debouncedSearch, page))` with search state + useDebounce
- Columns for AdminTable: username (text, read-only), email (text, read-only), role (display: badge; edit: dropdown with USER/ADMIN options), createdAt (formatted date, read-only)
- Override `onSave` to call `updateUserRole(id, newRole)` mutation -- only the role is editable
- No `onCreate` (users are created via Keycloak, not admin panel) -- hide the "+ Add" button by not passing `onCreate`
- `onDelete` calls `deleteUser(id)` mutation
- After mutation success: `queryClient.invalidateQueries({ queryKey: adminQueries.all() })`
- Also invalidate user-facing queries if they exist (no user-facing user list currently, so just admin keys)
- Pagination: "Load more" button if `!data.last`, incrementing page state
- Search placeholder: "Search users..."

**BlockedIpSection.tsx:**
- Uses `useQuery(adminQueries.blockedIps())`
- Columns: ipAddress (text input, editable on create only), reason (text input, optional), createdAt (date, read-only)
- `onCreate` calls `addBlockedIp({ ipAddress, reason })` mutation
- No `onSave` for existing rows (IPs are add/remove only) -- disable edit button or don't pass `onSave`
- `onDelete` calls `removeBlockedIp(id)` mutation
- IP validation: basic regex check `/^\d{1,3}(\.\d{1,3}){3}(\/\d{1,2})?$/` shown as inline error if invalid on save attempt
- Search placeholder: "Search IPs..."
- After mutation: invalidate `adminQueries.blockedIps()` query key

**ConfigSection.tsx:**
- Uses `useQuery(adminQueries.config())`
- Columns: key (text input -- editable ONLY during creation, read-only after save per research recommendation), value (text input, always editable)
- `onCreate` calls `createConfig({ key, value })` mutation
- `onSave` calls `updateConfig(id, { value })` mutation (only value changes)
- `onDelete` calls `deleteConfig(id)` mutation
- Search placeholder: "Search config..."
- After mutation: invalidate `adminQueries.config()` query key
