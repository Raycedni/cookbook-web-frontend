**AdminTagTree.tsx** -- Full editable tag tree component:

**Data fetching:**
- Uses `useQuery(adminQueries.tags())` to get flat `Tag[]`
- Converts to tree using `buildTagTree(tags)` from recipe types
- Also keeps flat tag list available for merge target selection

**Tree rendering:**
- Recursive `TagTreeNode` component that renders each node with indentation
- Each node shows: expand/collapse chevron (if has children), tag name, action buttons (edit, merge, delete)
- Indentation: `pl-{level * 6}` or similar nesting via margin
- Node styling: `py-2 px-3 rounded-lg hover:bg-white/5` with `flex items-center gap-2`

**Inline rename:**
- State: `editingId: number | null`, `editName: string`
- Click edit (Pencil icon) on a node: sets editingId, shows text input replacing name text with Save (Check) / Cancel (X) buttons
- Save calls `updateTag(id, { name: editName })` mutation
- Cancel clears editingId

**Create new tag:**
- "+ Add Tag" button at top of tree
- Opens an inline form: name text input + parent dropdown (select from flat tag list, with "None / Root" option) + Save/Cancel
- State: `creatingNew: boolean`, `newTagName: string`, `newTagParentId: number | null`
- Save calls `createTag({ name, parentId })` mutation

**Delete:**
- Click delete (Trash2 icon) on a node: `window.confirm('Delete tag "{name}"? This cannot be undone.')`
- If confirmed, calls `deleteTag(id)` mutation
- Note: deleting a parent with children -- the backend decides behavior (typically cascades or re-parents). UI just sends delete.

**Drag-and-drop move:**
- Use `DndContext` from `@dnd-kit/core` (NOT sortable -- tree reorder needs custom drop logic per Pitfall 5)
- Each tree node is both draggable (`useDraggable`) and droppable (`useDroppable`)
- `PointerSensor` with `distance: 8` (same as Phase 4 for click-vs-drag differentiation)
- `DragOverlay` renders a ghost of the dragged tag name in a GlassPanel
- On `onDragEnd`: if `over` exists and `over.id !== active.id`:
  - Prevent dropping a node into its own subtree (walk up parent chain to check)
  - Call `updateTag(activeId, { parentId: overId })` mutation to move the tag
- Drop target highlighting: `bg-accent/10 ring-1 ring-accent/30` when a draggable is over a valid target
- To drop as root: provide a "Root level" drop zone at the bottom of the tree

**After ANY mutation:**
- Invalidate `adminQueries.tags()` (admin cache)
- Invalidate `['tags']` (user-facing tag queries for recipe browsing -- Pitfall 3)

**Loading state:** Skeleton tree (3-4 indented skeleton lines)

**Empty state:** EmptyState with Tags icon and "No tags yet. Add your first tag."

**Wire route** (`admin/tags.tsx`):
- Replace scaffold with AdminTagTree
- Page wrapper: `<div><h1 className="text-2xl font-bold text-white mb-6">Tag Management</h1><AdminTagTree /></div>`

**TagMergeModal.tsx:**
- Props: `{ sourceTag: Tag; allTags: Tag[]; onConfirm: (targetId: number) => void; onClose: () => void }`
- Modal overlay: fixed inset-0, backdrop bg-black/60, centered GlassPanel with max-w-md
- Content:
  1. Title: "Merge Tag"
  2. Source display: "Merge '{sourceTag.name}' into..."
  3. Target selection: search input (with useDebounce) filtering `allTags` (excluding sourceTag and its descendants). Render filtered tags as a selectable list below the search input. Selected target highlighted with accent border.
  4. Once target selected, attempt to show affected recipe count. Try fetching recipe count from an endpoint like `GET /admin/tags/{sourceId}/merge-preview?targetId={targetId}`. If endpoint doesn't exist (404), fall back to showing just the confirmation without count.
  5. Confirmation text: "This will move all recipes tagged '{sourceTag.name}' to '{targetTag.name}' and delete '{sourceTag.name}'." (Add "Affects X recipes." if count available.)
  6. Buttons: "Cancel" (ghost button) and "Merge" (accent button, disabled until target selected)
- On confirm: calls `onConfirm(targetId)`

**Descendant exclusion for merge targets:**

- Filter allTags to exclude sourceTag.id and all descendant IDs (can't merge a parent into its own child)
